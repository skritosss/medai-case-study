# Prototype Overview

The private MedAI prototype archive was reviewed locally as evidence of implementation work. Only safe, high-level metadata is documented here.

## Prototype Type

- Flutter / FlutterFlow mobile application prototype.
- Mobile-first product workflow.
- Structured navigation built with `go_router`.
- Local state and UI components generated from a FlutterFlow-style project.

## Publicly Safe Evidence

| Evidence | Public summary |
| --- | --- |
| Framework | Flutter / Dart |
| App builder pattern | FlutterFlow-generated structure |
| Platforms | iOS, Android, and web project folders present |
| Main route system | `go_router`-based navigation |
| Screen count | multiple workflow screens plus reusable components |
| Assets | app icons, logo assets, images, fonts |
| Source code | intentionally private |

## Sanitized Screen Map

The reviewed prototype contains screens corresponding to:

- welcome / onboarding;
- home / workflow entry;
- state selection;
- list selection;
- detail review;
- structured numeric input;
- supporting informational modals;
- duplicated / experimental workflow variants.

This public case study does not expose clinical content, sensitive UI copy, private data, or decision logic.

## Why Not Publish the Source

Publishing the full archive would expose implementation details and product logic that are not needed to demonstrate the work publicly. The goal of this repository is to show product and architecture thinking while keeping sensitive product material private.
